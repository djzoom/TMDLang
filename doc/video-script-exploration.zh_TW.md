## 二、TMDLang 的核心架構特點

TMDLang（Timebase MarkDown Language）的設計初衷是「**基於時間軸的事件流描述語言**」，核心要素：

```
段落名稱:執行者@[|時間點|]{
    <計時單位*>
    執行內容
}
```

- **時間軸驅動**：所有內容都有精確的時間定位（小節/拍/十六分音符級別）
- **多軌並行**：多個段落（Paragraph）可在同一時間軸上疊加，各有獨立偏移
- **結構編排**：`-> intro -> A -> B -> ending ->#` 定義宏觀流程
- **動態參數**：速度 `{!=145}`、調性 `{?+5}`、拍號 `{<3/4>}` 可在流程中即時改變
- **純文字、VCS 友好**：天然適合 Git 版控與協作

## 三、成為短視頻/視頻輔助腳本的可行性分析

### 3.1 天然優勢

#### (a) 精確的時間軸體系
TMDLang 已具備從全音符到三十二分音符的時間精度（`<1*>` ~ `<32*>`），在 BPM=120 時：
- `<16*>` = 每個單位約 125ms，足以描述視頻剪輯中的精確時間點
- `<4*>` = 每個單位約 500ms，適合描述字幕與畫面切換

這天然適合作為影片時間軸的標記語言。

#### (b) 多軌同步結構
現有的多樂器段落機制可直接映射為視頻中的多軌道概念：

```
場景A:畫面@|0|{  ... }      → 視頻軌
場景A:字幕@|0|{  ... }      → 字幕軌
場景A:BGM@|0|{   ... }      → 背景音樂軌
場景A:音效@|+2|{ ... }      → 音效軌（延遲 2 小節入場）
```

#### (c) 宏觀流程控制
`-> hook -> verse -> chorus -> bridge ->#` 的結構，可直接對應短視頻常見敘事結構：

```
-> 開場 -> 問題 -> 轉折 -> 高潮 -> 結尾 ->#
```

#### (d) 純文字 + Git 版控
影片腳本的迭代修改、多人協作、歷史回溯，都能通過 Git 自然實現。

### 3.2 需要擴展的部分

#### (a) 時間表示的泛化
音樂語境下時間以「小節/拍」為單位，視頻語境需要支援絕對時間：

```
// 提案：新增絕對時間模式
<T:mm:ss.ms*>        // 以「分:秒.毫秒」為基礎單位
<T:frame@30fps*>     // 以「幀@幀率」為基礎單位

// 或保持 BPM 體系但增加時間戳標記
/*@00:15.500*/ [場景切換]
```

#### (b) 視覺指令擴展
音符 `1-7` 在視頻語境中需要替換為視覺事件描述：

```
場景A:畫面@|0|{
    <T:1s*>
    """
    [特寫] 主角面部
    [推鏡] 拉遠至全景
    [切] 空鏡頭-城市夜景
    [疊化] 回憶畫面
    """
}
```

現有語法文件已提到未來希望能記載 **Show Program**：
> 希望未來可以據此記載並生成整個 Show Program 的各種需求與協作檔案。

#### (c) 資源引用機制
需要能引用外部素材文件：

```
~assets: "./footage/"
~audio:  "./audio/"

場景A:畫面@|0|{
    <T:1s*>
    $footage/scene01.mp4[00:05-00:12]
    $footage/scene02.mp4[00:00-00:08]
}
```

#### (d) 字幕/文字層語法

```
場景A:字幕@|0|{
    <T:0.5s*>
    -- -- "你好世界" -- -- -- "歡迎來到" "TMDLang" --
}
```

利用現有的休止符 `-` 和引號字串來表達字幕的出現與消失。

### 3.3 短視頻腳本的完整範例構想

```
::VIDEO::
** 產品介紹短片 **
! = 120
<4/4>

hook:畫面@|0|{
    <T:0.5s*>
    """
    [特寫] 產品外觀旋轉
    [切] 使用場景-辦公桌
    [切] 使用場景-咖啡廳
    """
}
hook:字幕@|0|{
    <T:0.5s*>
    -- "你還在為___煩惱嗎？" -- -- --
}
hook:BGM@|0|{
    <4*>
    [1] [6m] [4] [5]
}

pain:畫面@|0|{
    <T:1s*>
    """
    [中景] 痛點演示1
    [中景] 痛點演示2
    [特寫] 表情反應
    """
}

solution:畫面@|0|{
    <T:0.5s*>
    """
    [全景] 產品登場
    [特寫] 核心功能演示
    [分屏] 對比效果
    """
}

cta:字幕@|0|{
    <T:1s*>
    "立即購買" "限時優惠" "點擊連結"
}

-> hook -> pain -> solution -> cta ->#
```

## 四、實現路徑建議

### 第一階段：語法擴展
1. 新增 `::VIDEO::` 檔頭類型（與現有 `::SCORE::` 並列）
2. 支援絕對時間模式 `<T:...>`
3. 引入三引號文字塊 `"""..."""` 描述視覺內容
4. 資源引用語法 `$path` 或 `~assets`

### 第二階段：解析器擴展
1. 在 TMDScanner.py 中新增 VIDEO 模式解析
2. C++ TMDLang++ 同步支援
3. 生成時間軸 JSON（可被 FFmpeg/DaVinci Resolve/剪映 等工具導入）

### 第三階段：渲染/導出
1. 生成 SRT/ASS 字幕文件
2. 生成 EDL (Edit Decision List) 或 XML 時間軸（Final Cut Pro / Premiere 格式）
3. 基於 Cairo 渲染可視化時間軸預覽圖
4. 與 FFmpeg 整合實現自動剪輯原型

## 五、結論

TMDLang 從設計哲學上就是一門**時間軸驅動的多軌事件描述語言**，它的核心架構——時間基底、多軌並行、結構編排、動態參數——與視頻腳本的需求有著高度的結構同構性。

**不需要推倒重來**，只需在現有語法體系上進行有針對性的擴展（絕對時間、視覺指令、資源引用），TMDLang 就有潛力成為一種獨特的、Git 友好的、精確到幀的視頻輔助腳本語言。

這在當前短視頻創作工具鏈中是一個空白：大多數腳本工具要麼是純文字（沒有時間軸概念），要麼是圖形化時間軸（無法版控協作）。TMDLang 有機會佔據這個交叉點。
